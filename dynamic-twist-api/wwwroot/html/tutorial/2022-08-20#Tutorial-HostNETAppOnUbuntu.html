The Tutorial

As I am thinking about this tutorial, it seems that it is going to be a long one. I’ll have to split it:

  • Setting up VPS: Install Nginx and .NET
  • Publishing .NET app
  • Transfer published files to VPS
  • Create a background service in Ubuntu
  • Configure Nginx

Setting up VPS: Install Nginx and .NET

As we’re getting brand new machine with nothing installed. In order to install Nginx you should run following commands:

sudo apt update
sudo apt install nginx

Since these are our first commands it’s recommended to run an update to get latest packages. Latter command is for installing Nginx itself.

Next you’ll want to check the status of Nginx, to make sure it’s running.

systemctl status nginx

When you run this command it should result in status check of Nginx process, where the status is running.

Now what is left is to install .NET SDK and runtime which is necessary for our .NET app to run on Ubuntu. Note that you’ll want to install version of .NET that corresponds to your app’s .NET version. We’ll do so with following commands:

sudo apt-get update
sudo apt-get install -y dotnet6
sudo apt-get install -y aspnetcore-runtime-6.0

Now we’ve had an initial setup of our VPS.

Publish .NET app

Going back to our Windows environment we’ll want to publish our .NET app that will contain runnable content for our VPS. If you already have .NET SDK on Windows (which you should), you’ll have .NET CLI alongside it. Considering that you have those on your machine already, publishing is straightforward.

You should open your cmd (Windows key + R, type in cmd) and navigate to projects folder using this command:

cd <project folder path>

Which might look something like this:

cd C:\Users\Bob\Documents\source\repos\Project1

After you’ve navigated, just type:

dotnet publish

Which will create a publish folder in C:\Users\Bob\Documents\source\repos\Project1\bin\Debug\net6.0, assuming that your .NET version is 6.

Transfer published files to VPS

Now we’ll need an FTP (File transfer protocol) client. We’ll be using WinSCP.

Setting up WinSCP is a theme for another time. Important thing is to setup authentication with your VPS, which can be done via SSH key that is connected to your Linux user.

Afterwards, you’ll want to transfer your publish folder to a directory in Ubuntu. Recommended path is /var/www.

Create a background service in Ubuntu

Now you’ll want to navigate to /etc/systemd/system/ and create a file. We’ll run the following command:

sudo nano kestrel-Project1.service

Which will create a file with .service extension. A text editor will open and then we’ll paste this data inside:

[Unit]
Description=Example .NET Web API App running on Ubuntu
[Service]
WorkingDirectory=/var/www/publish
ExecStart=/usr/bin/dotnet /var/www/publish/Project1.dll
Restart=always
# Restart service after 10 seconds if the dotnet service crashes:
RestartSec=10
KillSignal=SIGINT
SyslogIdentifier=dotnet-example
User=YourUser
Environment=ASPNETCORE_ENVIRONMENT=Production
Environment=DOTNET_PRINT_TELEMETRY_MESSAGE=false
[Install]
WantedBy=multi-user.target

This file will create a running service that will be running in the background. Important points to mention here are:

  • ExecStart command specifies where the .dll file that needs to be run is located.
  • Restart specify whether to restart the service or not if the service crashes.
  • User defines which user will be running this service.
  • Environment specifies environment variables for the running instance of the app.

After saving the file you’ll want to run the service with this command:

sudo systemctl start kestrel-Project1.service

Now the service is running, which we can check with this command:

sudo systemctl status kestrel-Project1.service

Which should note that our service is active (running).

Configure Ngnix

Last thing is to configure nginx config file.

First, navigate to config file directory:

cd /etc/nginx/conf.d

Then we’ll want to disable default config file:

sudo mv default.conf default.conf.disabled

And we’ll want to create a new config file:

sudo nano Project1.conf

And paste this content:

server {
  listen 80;
  listen [::]:80;
  server_name <your_ip_address>;
  location {
    proxy_pass http://localhost:<port>;
  }
}

We’ll save the changes and we’re good to go!

Conclusion

Last thing that we need to do is restart our Nginx with the command:

sudo systemctl restart nginx

And that’s about it! Now your VPS is hosting .NET app which you can access by typing in VPS’s IP address in browser.
